# Training Guide: Spatio-Temporal Earthquake Precursor Model

This guide explains how to train the spatio-temporal earthquake precursor model using the complete pipeline with stage-wise learning.

## Overview

The training pipeline implements a progressive learning approach across three stages:

1. **Stage 1**: Binary precursor detection (Precursor vs Solar Noise)
2. **Stage 2**: Magnitude classification with Focal Loss
3. **Stage 3**: Localization estimation (Azimuth + Distance)

## Prerequisites

### Data Requirements

1. **Scalogram Data**: Processed scalogram files from `scalogramv3` folder
2. **Metadata**: Master metadata CSV from data auditor (`master_metadata.csv`)
3. **Station Coordinates**: Station location data (`lokasi_stasiun.csv`)
4. **Tensor Data**: 5D tensors generated by tensor engine (HDF5 format)

### System Requirements

- Python 3.8+
- PyTorch 1.9+
- CUDA-capable GPU (recommended)
- Sufficient RAM for data loading (16GB+ recommended)

## Quick Start

### 1. Prepare Tensor Data

First, process your scalogram data into 5D tensors:

```bash
# Run tensor engine to create HDF5 files
python src/preprocessing/tensor_engine.py \
    --scalogram-path /path/to/scalogramv3 \
    --metadata-path outputs/data_audit/master_metadata.csv \
    --output-dir outputs/tensors \
    --stations ALR TND PLU GTO LWK GSI LWA SMI \
    --components H D Z \
    --target-shape 224 224
```

This creates:
- `outputs/tensors/train_original.h5`
- `outputs/tensors/train_cmr.h5` (with Common Mode Rejection)
- `outputs/tensors/test_original.h5`
- `outputs/tensors/test_cmr.h5`

### 2. Train the Model

Run the complete training pipeline:

```bash
# Train with default configuration
python train_model.py \
    --train-data outputs/tensors/train_cmr.h5 \
    --test-data outputs/tensors/test_cmr.h5 \
    --metadata outputs/data_audit/master_metadata.csv \
    --station-coords awal/lokasi_stasiun.csv \
    --config configs/training_config.yaml \
    --experiment-name earthquake_precursor_v1

# Train with custom settings
python train_model.py \
    --train-data outputs/tensors/train_cmr.h5 \
    --test-data outputs/tensors/test_cmr.h5 \
    --metadata outputs/data_audit/master_metadata.csv \
    --batch-size 32 \
    --num-workers 8 \
    --device cuda \
    --stages 1 2 3
```

### 3. Monitor Training

Training progress is logged to:
- Console output
- `outputs/training/{experiment_name}/training.log`
- TensorBoard: `outputs/training/{experiment_name}/tensorboard/`

View TensorBoard:
```bash
tensorboard --logdir outputs/training/{experiment_name}/tensorboard/
```

## Detailed Configuration

### Training Configuration File

The training configuration is defined in YAML format. See `configs/training_config.yaml` for the complete example.

#### Model Configuration
```yaml
model:
  n_stations: 8                    # Number of stations
  n_components: 3                  # Number of components (H, D, Z)
  efficientnet_pretrained: true    # Use pretrained EfficientNet
  gnn_hidden_dim: 256             # GNN hidden dimension
  gnn_num_layers: 3               # Number of GNN layers
  dropout_rate: 0.2               # Dropout rate
  magnitude_classes: 5            # Number of magnitude classes
```

#### Data Configuration
```yaml
data:
  batch_size: 16                  # Batch size
  num_workers: 4                  # Data loader workers
  pin_memory: true                # Pin memory for GPU
  load_in_memory: false           # Load all data in RAM
  magnitude_bins: [4.0, 4.5, 5.0, 5.5, 6.0]  # Magnitude class boundaries
```

#### Stage-Specific Configuration
```yaml
stage_1:
  epochs: 30                      # Number of epochs
  patience: 10                    # Early stopping patience
  train_backbone: true            # Train EfficientNet backbone
  train_gnn: true                 # Train GNN layers
  
  optimizer:
    type: AdamW                   # Optimizer type
    lr: 1.0e-4                    # Learning rate
    weight_decay: 1.0e-4          # Weight decay
  
  scheduler:
    type: CosineAnnealingLR       # Scheduler type
    T_max: 30                     # Cosine annealing period
    eta_min: 1.0e-6               # Minimum learning rate
```

### Loss Configuration
```yaml
loss_weights:
  binary_weight: 1.0              # Binary classification weight
  magnitude_weight: 1.0           # Magnitude classification weight
  localization_weight: 1.0        # Localization weight
  focal_alpha: 1.0                # Focal loss alpha
  focal_gamma: 2.0                # Focal loss gamma
```

## Training Stages

### Stage 1: Binary Precursor Detection

**Objective**: Distinguish earthquake precursors from solar noise

**Architecture**: 
- EfficientNet backbone + GNN fusion + Binary head
- Binary cross-entropy loss
- Confidence estimation

**Key Metrics**:
- Binary accuracy
- Precision/Recall/F1-score
- AUC-ROC

### Stage 2: Magnitude Classification

**Objective**: Classify earthquake magnitude into discrete classes

**Architecture**:
- Previous components + Magnitude head
- Focal loss for class imbalance
- Attention mechanism across stations

**Key Metrics**:
- Classification accuracy
- Per-class precision/recall
- Magnitude regression MAE

### Stage 3: Localization Estimation

**Objective**: Estimate earthquake azimuth and distance

**Architecture**:
- All previous components + Localization head
- Circular regression for azimuth
- Log-scale distance estimation
- Uncertainty quantification

**Key Metrics**:
- Azimuth error (degrees)
- Distance relative error
- Uncertainty calibration

## Advanced Usage

### Custom Training Script

For more control, create a custom training script:

```python
import torch
from src.models.spatio_temporal_model import create_model
from src.training.dataset import create_data_loaders
from src.training.trainer import SpatioTemporalTrainer

# Setup
device = 'cuda'
config = {...}  # Your configuration

# Create data loaders
train_loader, test_loader, dataset_info = create_data_loaders(
    train_hdf5_path='path/to/train.h5',
    test_hdf5_path='path/to/test.h5',
    metadata_path='path/to/metadata.csv',
    **config['data']
)

# Create model
model = create_model(config=config['model'])

# Create trainer
trainer = SpatioTemporalTrainer(
    model=model,
    train_loader=train_loader,
    val_loader=test_loader,
    device=device,
    output_dir='outputs/custom_training'
)

# Train progressively
training_history = trainer.train_progressive(config)

# Evaluate
test_metrics = trainer.evaluate_model(test_loader)
```

### Resume Training

Resume from a checkpoint:

```bash
python train_model.py \
    --train-data outputs/tensors/train_cmr.h5 \
    --test-data outputs/tensors/test_cmr.h5 \
    --metadata outputs/data_audit/master_metadata.csv \
    --resume-from outputs/training/experiment_20240415_120000/best_stage_2.pth \
    --stages 3  # Continue with stage 3 only
```

### Evaluation Only

Evaluate a trained model without training:

```bash
python train_model.py \
    --train-data outputs/tensors/train_cmr.h5 \
    --test-data outputs/tensors/test_cmr.h5 \
    --metadata outputs/data_audit/master_metadata.csv \
    --evaluate-only \
    --resume-from outputs/training/best_model/best_stage_3.pth
```

## Troubleshooting

### Common Issues

1. **CUDA Out of Memory**
   - Reduce batch size
   - Reduce model dimensions
   - Use gradient checkpointing

2. **Slow Data Loading**
   - Increase `num_workers`
   - Use `load_in_memory=true` if you have enough RAM
   - Check disk I/O performance

3. **Poor Convergence**
   - Adjust learning rates
   - Check data quality and preprocessing
   - Verify loss weights balance

4. **NaN Losses**
   - Check for invalid data (inf, nan)
   - Reduce learning rate
   - Add gradient clipping
